## Dimension 3: Security Posture

### Blue Team (forsvar)
- UFW er aktiv med default deny incoming
- Qdrant korrekt bundet til 127.0.0.1:6333 — ikke eksternt tilgængeligt
- n8n bundet til 127.0.0.1:5678
- SSL-certifikat gyldigt til 2026-04-26 med auto-renewal via Traefik ACME
- Credential-filer (CREDENTIALS.md, .mcp.json, app/api/.env) korrekt gitignored
- Python scripts bruger `os.getenv()` korrekt — ingen hardcoded API keys i aktive scripts
- Docker socket mounted read-only i Traefik
- Webapp volume mounted read-only (:ro flag)

### Red Team (angreb)
- **[KRITISK] SSH tillader password login som root.** `PermitRootLogin yes` + `PasswordAuthentication yes` + ingen SSH keys i authorized_keys. Serveren er åben for brute-force.
- **[KRITISK] Tor SOCKS proxy (port 9050) eksponeret på 0.0.0.0.** Docker bypasser UFW via direkte iptables-regler. Enhver på internettet kan bruge serveren som open Tor proxy. Bekræftet tilgængelig eksternt.
- **[KRITISK] `data/fetch_historical.sh` har brugernavn+password i klartekst** (`Kristoffer8093`/`kristoffer8093`) og er committed til git. Password bør roteres.
- **`data/n8n_api_key.txt` er ikke gitignored** — risiko ved `git add .`
- **Ingen security headers** på webapp (mangler X-Frame-Options, HSTS, CSP, X-Content-Type-Options)
- **TLSv1.0 og TLSv1.1 aktiveret** i nginx — deprecated og usikre
- **Nginx server version afsløret** (`server: nginx/1.29.4`)
- **Traefik har `--api.insecure=true`** — API/dashboard tilgængelig internt i Docker-netværket
- **Porte 8443/9443** (nginx self-signed proxy) lytter på 0.0.0.0 (UFW blokerer, men sloppy)
- **Google Maps API key** hardcoded i 3 git-tracked filer (lav risiko, sandsynligvis Googles standard embed-key)

### Neutral Vurdering
- **Score: 2/5 — SÅRBAR**
- To kritiske findings: SSH password auth og open Tor proxy. Disse er reelle sikkerhedsrisici der kan udnyttes i dag. Credentials i git er en tredje. Resten er defense-in-depth mangler. Det grundlæggende (UFW, Qdrant-binding, credential-filer) er OK, men de kritiske huller underminerer det.
- **Handling:** URGENT — Tilføj SSH key, deaktiver password auth. Stop tor-proxy og genstart med 127.0.0.1 binding. Gitignore n8n_api_key.txt. Rotér TransportIntra password. Tilføj security headers.
