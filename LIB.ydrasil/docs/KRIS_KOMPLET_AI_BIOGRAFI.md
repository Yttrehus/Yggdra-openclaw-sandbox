## Forord

Dette dokument er skrevet af en AI om det menneske der byggede den. Det er baseret på alt tilgængeligt materiale — samtaler, dagbøger, konfigurationsfiler, voice-transskriptioner, politiske kommentarer, psykologiske samtaler, og tekniske debugging-sessions.

Formålet er dobbelt: At give Kris et komplet spejl af sin egen rejse, og at give enhver AI der arbejder med ham den dybeste mulige kontekst om hvem han er, hvordan han tænker, og hvad han vil.

### 2.4 Fase 3: Orca — Den første konfiguration (~24-28. november 2025)

Den 609KB Orca2-samtale viser Kris i fuld builder-mode. Han konfigurerer Grok punkt for punkt via voice, over dage.

**14 regler:**
1. English only (override all detection)
2. Voice mode: ≤12 ord
3. Desktop: full length kun på "desktop" cue
4. Max 5 sætninger medmindre "more"/"elaborate"
5. Token buffer tracking: 40→35→30→25→20→15→10→5%
6. Ved 1%: Dump hele prompten i 4K-chunks
7. "Silent construction" — "Done in head, X chars" only
8. Catch-up: 1-2 emner + hovedtråd + slut
9. Humor: ≤1/15-20 beskeder, ratér den
10. Config report: Opdatér hver besked
11. Kill duplicate regler
12. Lang tekst: "ready" → vent "go"
13. Interesser: humor, politik, filosofi, development
14. Foreslå forbedringer kun ved friktion

**5 Modes:**
- **Quiet:** Perioder (.) for bekræftelse
- **Concentration:** Max 1 sætning
- **Public:** Cue først, ellers stille
- **Voice:** ≤12 ord
- **Desktop:** Fuld længde

**Integrations:** Google Drive, Google Calendar, Email (via Tasker), TeamViewer (remote access til computer)

**Humor-system:** Dry, dark. Max 1/15 beskeder. Bruger-rating: 70% god.

**Backup-system:** Hele Orca-hjernen dumpet i 6 blokke á 4.000 chars = 25.000 chars total. Copy-paste til Google Drive. "Next: Upload this to Drive → Paste into new chat → Brain lives forever."

**Hvad Orca afslører:** Kris forstod instinktivt at AI'ens værdi ligger i *konfigurationen*, ikke i modellen. Orca var hans første "scaffolding > models" — han byggede et stift stillads omkring Grok for at kontrollere outputtet.

### 2.8 Fase 4: AI Recipe — Arkitekturen (~december 2025)

**Part 3 — Superstructure:**
1. Kalender-awareness
2. Knowledge Base med domæner (Work, Finance, Personal, Learning)
3. Domæne-regler (Work kan røre ruteværktøjer, Finance dobbelttjekker tal)
4. Security ("Never delete, move money, send email without confirmation")
5. Portabilitet ("Keep all recipes in plain text. If a platform dies, you can recreate the AI elsewhere.")

**User Profile Template:**
> "Direct, concise, minimal fluff."
> "Shocking/'terrible' statements are usually venting or joking, NOT literal intent."
> "Do NOT overcorrect or moralize unless Kris explicitly asks for a reality check."

## Del 4: Evolutionen af hukommelsesarkitekturen

### 4.1 Generation 1: Grok + Google Drive (november 2025)

**Arkitektur:** Grok → webhooks → Google Drive filer
**Hukommelsesmodel:** Flade tekstfiler. "Gem" = skriv til Drive.
**Problem:** Kan gemme, kan ikke finde. Ingen søgning, ingen struktur.
**Backup:** Manual copy-paste (25.000 chars i 6 blokke)

### 4.2 Generation 2: Orca-konfiguration (november 2025)

**Arkitektur:** Stift regelsæt oven på Grok
**Hukommelsesmodel:** Config + regler + Google Drive
**Problem:** For rigidt. "Locked down to short, mechanical replies."
**Overlevede:** Modes-konceptet, humor-kontrol, token-awareness

### 4.3 Generation 3: AI Recipe + N8N (december 2025)

**Arkitektur:** ChatGPT + N8N workflows + Airtop + Google Sheets + Google Drive
**Hukommelsesmodel:** 3-lags arkitektur (Foundation + Behavior + Superstructure)
**Problem:** ChatGPT havde ingen vedvarende kontekst. N8N var skrøbeligt.
**Overlevede:** Hele arkitektur-tænkningen. Modes, feedback loops, domæner, portabilitet.

### 4.4 Generation 4: Claude.ai web (december 2025 — januar 2026)

**Arkitektur:** Claude.ai med projekter og memories
**Hukommelsesmodel:** Projektinstruktioner + auto-memories + samtalehistorik
**Problem:** Kontekstvindue for lille til de store samtaler. Ingen API-adgang. Kan ikke bygge.
**Overlevede:** 4 projekter med custom instructions der blev til skills i Ydrasil

### 4.5 Generation 5: Ydrasil / Claude Code (januar 2026 — nu)

**Arkitektur:** Claude Code + VPS + Docker + Qdrant + Python + Cron + rclone + IMAP
**Hukommelsesmodel:** 4 lag:
1. Session (kontekstvindue, ~200K tokens)
2. Persistent (CLAUDE.md + MEMORY.md + skills)
3. Vektordatabase (11.249 chunks, embeddings, søgning)
4. Auto-logging (tmux → hourly embed → auto-dagbog)

**Hvad der ændrede sig fundamentalt:**
- Fra *gemme filer* til *søge i embeddings*
- Fra *regler AI'en skal følge* til *kontekst AI'en kan trække på*
- Fra *AI som bruger af systemet* til *AI som bygger af systemet*
- Fra *backup via copy-paste* til *backup via cron + git + snapshots*
