# 03. AUTOMATION_INDEX

Dette dokument fungerer som det centrale overblik over alle aktive automatiske processer, hooks og scripts i Yggdra-økosystemet. Formålet er gennemsigtighed og forebyggelse af teknisk gæld (cruft).

## 1. OpenClaw Hooks (PC-instans)

Disse scripts kører automatisk ved specifikke hændelser i OpenClaw-sessionen.

| Hook | Script | Trigger | Formål |
|------|--------|---------|--------|
| **SessionStart** | `scripts/session_start.sh` | Ved start af ny session | Injektion af aktuel status fra `CONTEXT.md` og `DAGBOG.md`. |
| **PreCompact** | `scripts/pre_compact.sh` | Før context compaction | Opdatering af `chatlog.md`, kørsel af Fact Extraction pipeline, Qdrant ingestion og Maintenance Audit. |
| **SessionEnd** | `scripts/session_end.sh` | Ved afslutning af session | Logning af episoden til `data/episodes.jsonl` og Notion Sync trigger. |

## 2. Videns-pipeline (VPS-instans)

Drifts-scripts der kører som cronjobs på VPS-serveren for at indsamle og processere viden.

| Script | Frekvens | Formål | Status |
|--------|----------|--------|--------|
| `ai_intelligence.py` | Daglig (06:30) | Indsamling af nyheder fra RSS, HN, Twitter, Arxiv. | **AKTIV** |
| `youtube_monitor.py` | Daglig (07:00) | Overvåger YouTube-kanaler for nye videoer. | **AKTIV** |
| `daily_sweep.py` | Daglig (08:00) | Genererer daglig digest og rydder op. | **AKTIV** |
| `heartbeat.py` | Hver time | Hurtige tjek for notifikationer/mentions. | **INAKTIV** (Afventer VPS sync) |

## 4. Cruft Management

Når en automatisk proces ikke længere skaber værdi, skal den deaktiveres.

- **Check-frekvens:** Hver 30. dag i forbindelse med "Retrospective" sessioner.
- **Kriterie for sletning:** Hvis output fra en proces aldrig citeres eller handles på i 4 uger.
