# VPS Recovery Guide (Akut)

Dette dokument bruges, når `maintenance_audit.py` melder om manglende pipeline-output (daily files).

## 1. Tjek Status (VPS)
Log ind på VPS og kør:
```bash
crontab -l
systemctl status cron
```

## 2. Genstart Pipelinens Hjerte (Self-healing)
Forsøg først vores nye watchdog, der forsøger at genstarte de mest kritiske jobs automatisk:
```bash
cd /root/Yggdra
python3 scripts/pipeline_watchdog.py
```

Hvis `daily_sweep.py` stadig ikke er kørt, kør den manuelt:
```bash
python3 scripts/daily_sweep.py
```

## 3. Sync til Sandbox
Når filer er genereret på VPS, skal de "skubbes" til sandbox via Git:
```bash
git add data/intelligence/
git commit -m "Manual sync: Restoration of daily data"
git push origin main
```

## 5. Forebyggelse (Watchdog)
Når systemet er oppe igen, bør du aktivere `scripts/pipeline_watchdog.py` i din crontab for at muliggøre autonom selv-healing:
```bash
# Kør tjek hver 6. time
0 */6 * * * /usr/bin/python3 /root/Yggdra/scripts/pipeline_watchdog.py >> /root/Yggdra/data/watchdog.log 2>&1
```
