# Structural Merge Guide: Upstream to Flat Taxonomy

Dette dokument guider gennem processen med at merge upstream (gammel struktur) ind i sandboxens flade taksonomi.

## 1. Problematik
Upstream bruger stadig `projects/2_research/`, mens denne sandbox bruger `LIB.research/` osv. En direkte merge vil skabe "file location" konflikter.

## 2. Løsningsmodel (Pull Upstream v1.2)
Brug det automatiserede værktøj:
```bash
./scripts/pull_upstream.sh
```

## 3. Manuel procedure (hvis automatisering fejler)
1. Fetch upstream data: `git fetch upstream`
2. Kør translator: `python3 scripts/merge_translator.py`
3. Forsøg merge igen: `git merge upstream/main`
4. Løs eventuelle indholdskonflikter i `CONTEXT.md` manuelt.

## 4. Efter merge
Kør altid audit for at sikre at data er landet det rigtige sted:
```bash
python3 scripts/maintenance_audit.py
```
